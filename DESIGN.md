# AIUX Design System (视觉与排版规范)

## 1. Visual Theme & Atmosphere

AIUX 的核心视觉基调是：**“专业克制且具有人文温度的轻质科技感”**。
我们的设计哲学是“让复杂的AI逻辑，在视线之外，无形同载”。界面和输出物不应该大声喧哗，而应该像一个克制、敏锐且富有同理心的人类专家。页面应当以充足的留白作为奢侈感的信号，将内容（策略洞察、体验亮点）推向视觉的绝对中心。

**Key Characteristics (核心特征):**
- **轻质科技感 (Lightweight Tech):** 放弃沉重的赛博朋克深色模式、霓虹线条和密集的网格背景。使用具有物理真实感的轻量级动效（如细腻的 3D 粒子流场或轻柔的磨砂玻璃透视）来隐喻 AI 的运算能力。
- **人文温度 (Humanistic Warmth):** 科技感通过“空间与留白”体现，温度通过“低饱和度的底色”传递。绝不使用冰冷的纯白 (`#FFFFFF`)。
- **无痕设计 (Invisible Interface):** 拒绝过度装饰，摒弃传统 UI 的粗重边框。用严格对齐的排版和克制的字号跳跃来构建秩序。
- **高对比锚点:** 在极度素雅的背景中，通过极少量的品牌红 (`#FF0217`) 作为交互的反馈锚点，克制且锋利。

## 2. Color Palette & Roles

严禁使用默认的纯白或高饱和度渐变。色彩体系以“纸张暖白+石墨黑”构建人文底色，以极少量的品牌红作为视觉锚点。

### Primary & Accent
- **AIUX Red** (`#FF0217`): **【高危预警色】** 极高饱和度的品牌色。绝对禁止大面积涂抹（全局使用占比不得超过 5%）。仅限用于：核心 CTA 按钮的交互悬停、关键数据强调、细线装饰。

### Surface & Background
- **Pearl White** (`#FAFAFA`): 全局默认背景色，带有极微弱的人文暖灰，避免纯白刺眼。
- **Subdued Gray** (`#F3F4F6`): 用于次要卡片或引用模块的背景底色，与主背景形成微妙的空间分离。
- **Frosted Glass** (`rgba(250, 250, 250, 0.8)`): 带有背景模糊 (`backdrop-filter: blur(12px)`) 的半透明白，用于悬浮导航或顶栏。

### Neutrals & Text
- **Graphite Black** (`#222428`): 全局正文与各级标题颜色，提供专业、清晰但不过分锐利的阅读体验。
- **Cool Stone** (`#6B7280`): 用于次要说明文字、时间戳或辅助信息。
- **Silver Fog** (`#9CA3AF`): 占位符文本、被禁用的状态或极弱的分割线。

## 3. Typography Rules

采用中西文混排的双字体系统，通过克制、严谨的字号跳跃构建专业感。

### Font Family
- **Alphanumeric (英文与数字)**: `Roboto` — 传达精准、理性的科技骨架。
- **Chinese (中文)**: `Source Han Sans SC` (思源黑体) — 现代、中性且高阅读性的排版基石。

### Hierarchy (层级量化表)

| Role | Size | Weight | Line Height | Letter Spacing | Notes |
|------|------|--------|-------------|----------------|-------|
| **H1 (一级标题)** | 36px | 500 | 1.3 | normal | 核心论点，必须简明扼要（15字内） |
| **H2 (二级标题)** | 24px | 500 | 1.4 | normal | 模块拆解标题 |
| **H3 (三级标题)** | 18px | 500 | 1.5 | normal | 卡片标题或段落重点 |
| **Body (正文)** | 14px | 400 | 1.6 | normal | 默认段落文本，必须保持 1.6 行高 |
| **Button (按钮)** | 14px | 500 | 1.0 | normal | 按钮标签文字 |
| **Caption (图注)** | 12px | 400 | 1.4 | normal | 使用 `Cool Stone` (#6B7280) |

## 4. Component Stylings

### Buttons (按钮)
- **Primary CTA**: 背景 `#222428`，文本 `#FAFAFA`，圆角 `4px`。当交互 (Hover) 时，边框或背景可透露极微量的 `#FF0217`。
- **Secondary CTA**: 幽灵按钮 (Ghost)，背景透明，边框 `1px solid #9CA3AF`，文本 `#222428`。

### Cards & Containers (卡片)
- **无边框设计**: 仅依靠背景色 (`#F3F4F6`) 与主背景分离，内部留出充足的 padding（如 `24px` 或 `32px`）。

### Inputs & Forms (表单)
- 背景 `#FAFAFA`，边框 `1px solid #9CA3AF`。聚焦 (Focus) 时，边框变为 `#222428`。圆角保持 `4px`。

## 5. Layout Principles

### Spacing System (间距系统)
- **基础网格**: `8px`。所有内边距 (Padding) 和外边距 (Margin) 必须是 8 的倍数（8, 16, 24, 32, 48, 64px）。
- **极度留白**: 模块与模块之间必须保持 `64px` - `80px` 的空间。一次只让用户聚焦一个核心观点。

### Border Radius Scale (圆角层级表)

| Value | Context |
|-------|---------|
| **0px** | 绝大多数非交互元素的默认状态。 |
| **4px** | 按钮、输入框、下拉菜单 (精准克制的交互感)。 |
| **8px** | 图片容器、大面积卡片底板。 |
| **50%** | 仅用于头像或轮播图的指示点。 |

## 6. Depth & Elevation

**禁止传统阴影 (No Shadows)**：彻底摒弃传统 UI 的弥散阴影或投影。深度只能通过背景色阶或透明度变化来实现。

| Level | Treatment | Use |
|-------|-----------|-----|
| **Level 0 (Flat)** | 无阴影，无边框 | 默认状态：底层画布，普通文本。 |
| **Level 1 (Subdued)**| 纯色背景 `#F3F4F6` | 内容卡片，区分主次信息的容器。 |
| **Level 2 (Frost)** | `rgba(250,250,250,0.8)` + `blur(12px)` | 悬浮导航栏、吸顶 Header。 |
| **Level 3 (Overlay)**| `rgba(34,36,40,0.4)` (遮罩) | 弹窗 (Modal) 或图片全屏预览时的底部遮罩。 |

## 7. Do's and Don'ts (体验护栏与去 AI 化禁忌)

### Do (必须执行)
- **情境针对性设计**: 布局必须紧扣商业场景（如智舱体验、五感创新）。
- **客观数据的视觉化**: 出现并列逻辑或数据对比时，强制渲染为具有极简科技感的表格或无边框信息流。
- **客观落地的文案**: 文本表述像资深咨询顾问，多用动宾短语，直接陈述“业务痛点”，用真实场景替代主观形容词。

### Don't (绝对禁止)
- **禁止烂大街 AI 字体**: 绝对禁止使用 `Inter`、`Arial` 或 `Space Grotesk`。
- **禁止廉价科技配色**: 绝对禁止在白色背景上使用生硬的“紫色/品红色渐变”或“蓝青色极光”，拒绝发光效果。
- **禁止可预测的布局**: 拒绝千篇一律的卡片瀑布流。
- **禁止 AI 八股文**: 绝对禁止生成“底层逻辑”、“赋能”、“在这个瞬息万变的时代”、“综上所述”等陈词滥调。
- **禁止低级科幻元素**: 拒绝插入发光的机械大脑、悬浮的二进制代码等图片。
- **禁止AI常规卡片样式**: 拒绝卡片样式采用同一个样式模版，如AI常用的，卡片左侧边缘一条细细的有色的长条。

## 8. Responsive Behavior (响应式断点)

| Breakpoint | Width | Key Changes |
|------------|-------|-------------|
| **Mobile** | `<768px` | 单列布局；H1 缩放至 `28px`；导航折叠为汉堡菜单；侧边距减小至 `16px`。 |
| **Tablet** | `768-1024px` | 允许 2 列布局；侧边距保持 `32px`。 |
| **Desktop**| `>1024px` | 内容最大宽度限制为 `1200px` 居中；模块间距放大至 `80px`。 |

## 9. Agent Prompt Guide (给 AI 生成器的指南)

### Quick Color Reference
- Background: "Pearl White (#FAFAFA)"
- Text: "Graphite Black (#222428)"
- Accent/CTA Hover: "AIUX Red (#FF0217)"
- Card/Surface: "Subdued Gray (#F3F4F6)"

### Execution Rules for Agents
- "当你生成产品调研报告或代码时，请使用 #FAFAFA 作为整体纸张底色，标题和正文使用 #222428。"
- "严禁使用 box-shadow 属性。深度仅通过背景色阶区分。"
- "排版必须极其空旷。所有中文强制应用 Source Han Sans SC 渲染，数字与英文应用 Roboto 渲染。"
- "如果使用到红色的强调色 (#FF0217)，确保它的视觉面积极小，通常只作为下划线、状态原点或 Hover 交互的反馈。"
